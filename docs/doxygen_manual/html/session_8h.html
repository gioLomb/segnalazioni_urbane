Descrizione dettagliata

In-memory session management backed by a hash table.

Sessions are identified by a random hex token stored in a browser cookie. Each session embeds the full User struct so route handlers can read role, username and city without an extra database round-trip.

Lifecycle: session_init() at startup, session_destroy_all() at shutdown.

bool session_create ( const User *  user,
char *  outToken 
)

Creates a new session for the given user and writes the token to outToken.

Generates a cryptographically random hex token and retries on the rare chance of a collision with an existing session.

Precondizione
session_init() has been called.
Parametri
userPointer to the authenticated User struct (copied in).
outTokenBuffer of at least TOKEN_HEX_LEN + 1 bytes.
Restituisce
true on success, false on allocation or collision failure.

Verifies a token and copies the cached User into outUser if valid.

Performs lazy eviction: expired sessions are removed on first access rather than by a background sweep.

Parametri
