Descrizione dettagliata

Generic fixed-size block allocator (slab allocator).

Manages many objects of the same size efficiently by allocating memory in large "chunks" and subdividing them into fixed-size slots. This reduces heap fragmentation and keeps alloc/free at O(1).

Memory layout of each internal block: [ Chunk* back-ptr ][ ... block_size bytes of user data ... ]

The hidden back-pointer enables O(1) deallocation without any extra metadata in the caller's struct.

◆ slab_pool_alloc()

void * slab_pool_alloc ( SlabPool *restrict  pool)

Allocates one block from the pool.

The returned memory is NOT zero-initialised; callers that require zeroing must do so themselves.

Precondizione
pool is initialised.
Postcondizione
On success, a pointer to exactly blockSize bytes is returned. The pool may internally allocate a new chunk if all existing chunks are full.
Parametri
poolPointer to the SlabPool.
Restituisce
Pointer to the allocated block, or NULL if out of memory.

poolPointer to the SlabPool to destroy.

◆ slab_pool_free()

void slab_pool_free ( SlabPool *restrict  pool,
void *restrict  ptr 
)

Returns a block to the pool for reuse.

Empty chunks are freed immediately, except the last one, which is retained to avoid allocation thrashing under low load.

Precondizione
ptr was returned by slab_pool_alloc() from the same pool.
Postcondizione
The block is pushed onto the chunk's free-list; the chunk may be freed if it becomes entirely empty.
Parametri
poolPointer to the SlabPool.
ptrPointer to the block to return.

Initialises the pool for blocks of the given size.

Allocates the first chunk immediately so the pool is ready for use on return.

Precondizione
pool != NULL, blockSize > 0, chunkCapacity > 0.
Postcondizione
The pool is ready; slab_pool_alloc() can be called immediately.
Parametri
poolPointer to the SlabPool to initialise.
blockSizeSize in bytes of each individual allocation.
chunkCapacityNumber of blocks to provision per chunk.
Restituisce
0 on success, -1 if the initial chunk allocation fails.
