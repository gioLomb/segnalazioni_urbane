Descrizione dettagliata

User management: registration, authentication, and role queries.

Passwords are stored as DJB2-derived hashes over a random salt. Legacy rows without a salt are supported transparently in user_authenticate().

Authenticates a user against stored credentials.

Supports both salted hashes (current) and legacy unsalted hashes transparently. Uses a constant-time comparison to prevent timing attacks.

Registers a new user with a randomly salted password hash.

Generates a random salt, computes hash(salt + plainPassword), and stores both in the database.

Parametri

cityMunicipality the user belongs to.

Restituisce
